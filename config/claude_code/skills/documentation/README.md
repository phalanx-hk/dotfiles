# ドキュメント作成

## 基本な考え方
- 書かれている内容が推測しやすい、簡潔で分かりやすい見出しをつける
- １つの見出しには１つのテーマを持たせる
- 記述を共通化し、必要な箇所から参照リンクを貼る

## 基本的なドキュメント作成の進め方

ドキュメントの種類や内容によって最適なドキュメント作成の進め方は異なりますが、以下の手順を参考にドキュメントを作成してみてください。

### 読み手とテーマを明確化

- 読み手とテーマが明確でないと、ドキュメントが大きくかつ複雑になるため、可能な限り絞り込むようにしましょう
- 読み手に関しては、以下３つに焦点を当て、ターゲットを絞り込むようにしましょう
    - 読み手の目的
    - 知識レベル
    - 立場

### テーマを階層構造に分解

- ドキュメントのテーマを複数のサブテーマで構成される階層構造に分解します
    - 例: xx機能を説明する: xx機能の概要、xx機能の設定方法、xx機能の利用方法、トラブルシューティング
- テーマを階層構造に分解することで、複雑な情報が論理だった構成となり、読み手が納得を得られるようなドキュメントになります
  - 全体から部分へ、概要から詳細へと段階的に情報を提供できるようになります
  - タイトルや見出しから内容を推測しやすくなります
- テーマの分割は「なぜ・何を・どうやって」の３つの観点で分割していくと効果的です
    - なぜ： なぜその企画が必要なのか
    - 何を： その企画で何を実現したいのか
    - どうやって： その企画をどのように実現するのか

### ドキュメントの構成を組む

- ドキュメントの階層構造の要素を見出しとしましょう
- 見出しの名前は「## 基本」を参考に名前をつけます

### 文章の構成を組む

- 見出しの中に書く文章の構成を組んでいきます
- いきなり文章を書こうとせず、その見出しで言いたいこと（topic sentence）を箇条書きで洗い出しましょう
- 言いたいことを書き出したら、言いたいこと同士の繋がりがよくなるように並べ替えます
- 言いたいこと同士の繋がりが悪ければ、繋がるように文を書き足します
- こうすることで見出しの中全体で言いたいことが表現され、読み手にとって分かりやすい文章になります
- 

### 文を書く

- １つ１つの言いたいことに対して、読み手が納得できるよう、その理由や説明を書き足します
- 理由や説明の書き方は以下の３パターンがあります
    - 理由を述べる（なぜなら〜だからです）
    - 説明を述べる（〜とは〜のことです）
    - 具体例を挙げる（例えば〜のような場合です）

### タイトルや見出しのリード文を書く
- タイトルや見出しの名前だけでは内容が把握しづらい場合、内容を補足するリード文を書きます
- 上記以外にも、前の見出しとの繋がりを良くできるため、そのケースでも必要に応じてリード文を書きましょう

## 分かりやすい文を書くコツ

### 重要なことから書く

- 情報に優先度をつけて、優先度が高いことから書きます
- 言いたいことをまず書き、その理由や説明を後から書く形にします。

### 読み手の視点で書く

- 読み手が著者の視点に置き換えて読むのはあまりにも非効率かつ読みづらいです
- 読み手の視点に立って、読み手が理解しやすいように書きます
- 例
  - 改善前：アプリの自動アップデートが有効になっていると、自動でアップデートの有無をチェックし、自動的にアプリをアップデートします
  - 改善後：アプリの自動アップデートを有効にすると、アプリが自動的に最新バージョンにアップデートされます

### 簡潔に書く

- できるだけ１つの分は短くなるようにしましょう
- １つの文には１つの情報のみ含まれているのが理想です
- ここでの情報は、ある特定の話題に関する事実、意見、説明、または指示を指します

- 例
  - 改善前：アプリの自動アップデートを有効にすると、アプリが自動的に最新バージョンにアップデートされ、常に最新の機能とセキュリティパッチを利用できます
  - 改善後：アプリの自動アップデートを有効にすると、アプリが自動的に最新バージョンにアップデートされます。これにより、常に最新の機能とセキュリティパッチを利用できます。

### 意味を明確にするために肯定形を使う

- 否定形は、意味が分かりづらくなることがあります
- 肯定的な表現ができる場合は、できるだけ肯定形を使うようにしましょう
- 例
  - 改善前：アプリのアップデートは放置したままにしないでください。
  - 改善後：アプリのアップデートはなるべく早く行ってください。
